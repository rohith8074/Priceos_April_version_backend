Reservation Agent Implementation Checklist

Scope based on PriceOS_Reservation_Agent_Spec_V2 (2).pdf and your goal: automate routine reservation communications with human-safe guardrails.

Use this as a build checklist. Mark each line complete as features go live.

1) Core Data Models (KB)

  • Create/verify schemas: listing_profile, reservation, guest_profile.
  • Create/verify conversation schemas: guest_thread, guest_message.
  • Add policy schema: comms_policy (tone, language, AI disclosure, thresholds).
  • Add ops_ticket schema with SLA/severity/status fields.
  • Ensure audit fields and timestamps exist on all records.

2) Tool Contracts (Agent Actions)

  • Implement listThreads and readThread.
  • Implement sendGuestMessage with strict validation and channel constraints.
  • Implement closeThread, createOpsTicket, escalateThread.
  • Implement secure send_access_details (no raw free-text secrets).
  • Implement sendUpsellOffer and logVoiceCall.

3) Safety & Routing Logic

  • Enforce comms_state hard gate (active/paused/syncing/disabled).
  • Add intent + sentiment + confidence classification before every action.
  • Implement confidence thresholds (auto-send vs draft vs escalate).
  • Implement mandatory high-risk escalation paths (refund/legal/safety/disputes).
  • Add human takeover/resume protocol for paused threads.

4) Proactive Lifecycle Automation

  • Booking confirmation sequence (T+0, T+24h, T-7d).
  • Pre-arrival sequence (T-48h access details, T-24h reminder, T-4h upsell).
  • In-stay sequence (check-in confirmation, mid-stay check).
  • Checkout/post-stay sequence (checkout + review nudges).
  • Queue proactive sends when not active; notify manager agent.

5) Omnichannel Integrations

  • Hostaway inbox/thread sync integration.
  • WhatsApp, email, SMS send + status callbacks.
  • Twilio voice inbound/outbound and transcript ingestion.
  • Payment/compliance handoff hooks (approval-required actions only).
  • Channel-specific policy enforcement (OTA constraints, cancellation contexts).

6) UI (Unified Inbox)

  • Thread list with status tabs, filters, unread indicators.
  • Thread detail timeline with intent/sentiment/confidence badges.
  • Draft card with approve/edit/reject/escalate.
  • Comms state control (Active/Paused) with confirmation modal.
  • Inline ops ticket status and escalation history visibility.

7) Structured Output & Observability

  • Emit and store BaseEnvelope cards for every action.
  • Implement envelope types: draft_reply, reply_sent, escalation, ticket_created, thread_closed, upsell_sent.
  • Add monitoring dashboards for response time, confidence, escalation, send failures.
  • Add alerting for policy violations (e.g., send while paused).
  • Ensure replay/debug trace for critical incidents.

8) QA & Go-Live

  • Convert spec Section 13 scenarios into automated test cases.
  • Run load and latency tests for message and voice paths.
  • Validate multilingual behavior against comms policy.
  • Perform shadow mode / pilot rollout before full auto-send.
  • Define rollback and incident response runbook.
Recommended rollout order: Core data + tools → safety routing → inbox approvals → proactive messaging → voice + advanced integrations → full KPI/QA hardening.
This reaches “automated receptionist” behavior for routine cases while keeping human control for risk cases.
